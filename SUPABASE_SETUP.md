# Supabase Integration Guide for Best Buy Finder

## Quick Setup

### 1. Create Supabase Account
- Go to [supabase.io](https://supabase.io) and sign up
- Create a new project
- Wait for project initialization

### 2. Copy Your Credentials
In **Project Settings → API**, find and copy:
- **Project URL** (looks like `https://xxxxx.supabase.co`)
- **anon public key** (starts with `eyJ...`)

### 3. Create Database Tables

Go to **SQL Editor** in your Supabase dashboard and run:

```sql
-- Table for tracking events (login, search, etc.)
CREATE TABLE app_events (
  id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  event_type TEXT NOT NULL,
  username TEXT,
  payload JSONB,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Table for cart snapshots
CREATE TABLE cart_snapshots (
  id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  username TEXT NOT NULL,
  items JSONB,
  total_amount NUMERIC,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Table for price alerts
CREATE TABLE price_alerts (
  id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  username TEXT,
  title TEXT,
  target_price NUMERIC,
  email TEXT,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
```

### 4. Enable Row Level Security (RLS)

For **development** (allow anyone to insert):

```sql
-- Enable RLS on all tables
ALTER TABLE app_events ENABLE ROW LEVEL SECURITY;
ALTER TABLE cart_snapshots ENABLE ROW LEVEL SECURITY;
ALTER TABLE price_alerts ENABLE ROW LEVEL SECURITY;

-- Create permissive policies for anonymous users
CREATE POLICY "allow_anon_insert_events" ON app_events FOR INSERT WITH CHECK (true);
CREATE POLICY "allow_anon_insert_carts" ON cart_snapshots FOR INSERT WITH CHECK (true);
CREATE POLICY "allow_anon_insert_alerts" ON price_alerts FOR INSERT WITH CHECK (true);
```

> ⚠️ **Production Note**: Use proper RLS policies that verify users before allowing insert/update.

### 5. Run Flutter App with Supabase

```bash
cd mobile_app

flutter run \
  --dart-define=SUPABASE_URL=https://YOUR_PROJECT.supabase.co \
  --dart-define=SUPABASE_ANON_KEY=YOUR_ANON_KEY
```

Replace:
- `YOUR_PROJECT` with your Supabase project name
- `YOUR_ANON_KEY` with the anon public key you copied

### 6. (Optional) Use Environment File

Create `.env` in `mobile_app/`:
```
SUPABASE_URL=https://YOUR_PROJECT.supabase.co
SUPABASE_ANON_KEY=YOUR_ANON_KEY
```

Then run with environment expansion (requires shell setup).

## Verification

### Check in Supabase Dashboard
1. Go to **Editor→ app_events / cart_snapshots / price_alerts**
2. You should see new rows appearing as you use the app
3. Use **SQL Editor** to query:

```sql
SELECT * FROM app_events WHERE event_type = 'search' ORDER BY created_at DESC LIMIT 10;
SELECT * FROM price_alerts WHERE username = 'your_username';
SELECT * FROM cart_snapshots WHERE username = 'your_username' ORDER BY created_at DESC LIMIT 5;
```

## Troubleshooting

| Problem | Solution |
|---------|----------|
| App doesn't connect to Supabase | Check credentials are correct in `--dart-define` or `.env` |
| "Permission denied" errors | Enable RLS policies (see step 4) |
| No data appearing in tables | Check Supabase credentials and ensure Flutter app is running with them |
| Tables don't exist | Run SQL creation script above in Supabase SQL Editor |

---

## Security Notes

- **Development Mode** (no Supabase): App works fully without cloud sync
- **API Keys**: The `anon` key is public (safe to use in client apps)
- **RLS Policies**: Always restrict insert/update/delete to authenticated users in production
- **Sensitive Data**: Don't store passwords or payment info in Supabase

---

## Production Deployment

1. **Update Flask backend** `apiBaseUrl` to your production server
2. **Create strict RLS policies** (don't allow anonymous inserts)
3. **Enable Supabase auth** (authenticates users before tracking)
4. **Set up backups** in Supabase dashboard
5. **Monitor usage** - Check Supabase storage/bandwidth limits

## Example: Advanced RLS Policy

```sql
-- Only allow authenticated users to insert their own events
CREATE POLICY "authenticated_events_insert" ON app_events
FOR INSERT
WITH CHECK (
  auth.uid()::text = username
);
```

This requires Supabase Auth to be enabled. For now, development mode uses anonymous inserts.
