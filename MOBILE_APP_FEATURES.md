## 🔄 Supabase Integration

### What Gets Synced to Supabase?

1. **Auth Events**
   - User login/logout state changes
   - Username and session status

2. **Search Tracking**
   - Search queries performed
   - Result counts
   - Search timestamps

3. **Cart Operations**
   - Cart snapshots (items, total amount)
   - Added when cart is updated

4. **Price Alerts**
   - Alert title, target price, email
   - When user sets price alerts

### How to Set Up Supabase

#### Option 1: With Supabase Cloud (Recommended)

1. Sign up at [supabase.io](https://supabase.io)
2. Create a new project
3. In your project settings, note:
   - **Project URL** (Supabase URL)
   - **Anon Key** (Public API key)

4. Create the required tables in SQL Editor:

```sql
-- App Events (for auth, search, tracking)
create table if not exists app_events (
  id bigint primary key generated by default as identity,
  event_type text not null,
  username text,
  payload jsonb,
  created_at timestamp with time zone default now()
);

-- Cart Snapshots
create table if not exists cart_snapshots (
  id bigint primary key generated by default as identity,
  username text not null,
  items jsonb,
  total_amount numeric,
  created_at timestamp with time zone default now()
);

-- Price Alerts
create table if not exists price_alerts (
  id bigint primary key generated by default as identity,
  username text,
  title text,
  target_price numeric,
  email text,
  created_at timestamp with time zone default now()
);
```

5. Enable RLS (Row Level Security) for development:
```sql
alter table app_events enable row level security;
alter table cart_snapshots enable row level security;
alter table price_alerts enable row level security;

-- Allow anonymous inserts
create policy "allow_anon_insert" on app_events for insert with check (true);
create policy "allow_anon_insert" on cart_snapshots for insert with check (true);
create policy "allow_anon_insert" on price_alerts for insert with check (true);
```

---

#### Option 2: Without Supabase (Development/Test)

If Supabase credentials are not provided, the app gracefully falls back to working with just Flask backend (all features still work).

### Running the App with Supabase

```bash
cd mobile_app

# With Supabase
flutter run \
  --dart-define=SUPABASE_URL=https://YOUR_PROJECT.supabase.co \
  --dart-define=SUPABASE_ANON_KEY=YOUR_ANON_KEY

# Without Supabase (development mode)
flutter run
```

## 🔗 Backend API Integration

All features connect to Flask backend endpoints:

| Feature | Endpoint | Method |
|---------|----------|--------|
| Get Products | `/api/products` | GET |
| Search Products | `/api/search?q=` | GET |
| Add to Cart | `/api/cart/add` | POST |
| Get Cart | `/api/cart` | GET |
| Optimize Cart | `/api/cart/optimize` | GET |
| Checkout | `/api/checkout` | POST |
| Get Orders | `/api/orders` | GET |
| Price History | `/api/price-history` | POST |
| Set Price Alert | `/api/set-alert` | POST |
| AI Summary | `/api/ai-summary` | POST |
| Register | `/api/register` | POST |
| Login | `/api/login` | POST |
| Logout | `/api/logout` | POST |

## 🚀 Running the App

### Prerequisites
- Flutter SDK 3.10+
- Android SDK / iOS SDK
- Supabase credentials (optional)

### Commands

```bash
cd mobile_app

# Get dependencies
flutter pub get

# Run on connected device
flutter run

# Run on emulator with Supabase
flutter run \
  --dart-define=SUPABASE_URL=https://YOUR.supabase.co \
  --dart-define=SUPABASE_ANON_KEY=YOUR_KEY

# Build APK
flutter build apk --release

# Build iOS
flutter build ios --release
```

## Support & Troubleshooting

- **App not connecting to backend?** Check `apiBaseUrl` in `app_config.dart`
- **Supabase errors?** Ensure tables are created and RLS policies allow anonymous inserts
- **Search not working?** Make sure Flask backend is running and SERPAPI_KEY is set
- **Cart not persisting?** Verify Flask session configuration
